35 // shared_cache considers 2 types of actors: consumers and producers
36 // this structure, expects there to be multiple consumers, but only 1 producer
37 // cache makes use of a key-value system to reduce duplicate/obsolete information
38 // key is unique, meaning when a new key-value pair is inserted,
39 // the old pair with the same key is obsolete
40 // because producers may reference some obsolete pairs, we handle obsolecse by shared_pointers to nodes
41 // obsolete nodes always link to a valid node or nullptr
42 // obsolete nodes have prev of null
43 // the producer determines a key-value relationship using hash function (hasher_)
44 // each consumer holds one shared_ptr to some node in the cache (content_)
45 // if the shared_ptr is null, the consumer is expected to consume from the start of the cache
46 // otherwise, the consumer consumes the immediate valid node referenced by the shared_ptr
